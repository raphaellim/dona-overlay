# Donation Overlay V1 Plus Sections

지금까지 수정된 1차 최종본에 기능을 추가한 버전입니다.

추가:
- control.html에서 노래방창 ON/OFF
- control.html에서 펀딩창 ON/OFF
- control.html에서 방송시간 타이머 ON/OFF
- overlay.html에 노래방창 / 펀딩창 / 상단 방송시간 타이머 표시
- 계좌후원 표시: 이름 ♥ 금액
- 금액별 하트 색상
- 계좌후원 금액 폰트 축소

유지:
- 수동 사운드 ALERT / 소리 재생
- 대기열 전송 → 순차 재생
- 반복 횟수 / 남은 횟수 표시
- monitor=1은 대기열 소비 안 함
- 후원 ALERT
- 크리에이터박스 OFF 상태에서도 ALERT 표시
- 공지 제목/내용/줄바꿈/색상 표시
- 크리에이터 후원 표시 ON/OFF

제외:
- 재전송/멈춤/claim/lease 기능은 제외했습니다.

적용:
1. ZIP 전체 덮어쓰기
2. GitHub Commit
3. Supabase SQL Editor에서 donation_overlay_v1_plus_sections.sql 실행
4. Render 재배포
5. control.html에서 설정 저장
6. overlay.html 캐시 방지 URL로 확인

추가 보정:
- ALERT가 공지박스/계좌후원박스 가로폭 안에서 중앙 표시되도록 수정
- OBS 세로 화면에서 ALERT가 옆으로 튀는 현상 완화


## ALERT 기준 박스 강제 보정

- 기존 `.option-alert { position:fixed; left:50%; top:50%; }` 규칙을 강제로 무력화했습니다.
- `#overlayAlertRoot`는 공지박스/계좌박스/크리에이터박스의 좌표와 크기를 따라갑니다.
- `#optionAlert`는 `#overlayAlertRoot` 안에서 `position:absolute`로 중앙 표시됩니다.


## 박스 스택 순서 수정

타이머/계좌후원/펀딩/공지/노래/크리에이터 박스가 전체 페이지 기준으로 뜨지 않도록,
모두 `.overlay-stack` 안으로 이동했습니다.

표시 순서:
1. 방송시간 타이머
2. 계좌후원
3. 펀딩
4. 공지
5. 노래방
6. 크리에이터


## 노래방 관리 기능 추가

control.html에 노래방 관리 섹션을 추가했습니다.

기능:
- 닉네임별 코인추가
  - 1절코인
  - 완곡코인
  - 스페셜코인
- 코인이 있는 닉네임만 선곡 가능
- 선곡 시 해당 코인 1개 자동 차감
- 선곡 리스트업
- 각 노래 상태 변경
  - 진행
  - 대기
  - 보류
  - 완료
  - 삭제
  - 변경
- overlay 노래방 박스에 진행중/예약순번/현재 코인보유/흐르는 공지 표시


## 요청사항 통합 수정

- 노래방 관리를 control.html에서 admin.html로 이동했습니다.
- 펀딩 관리는 control.html에 추가했습니다.
- 후원 입력 시 펀딩 선택/만원단위 반영을 추가했습니다.
- 펀딩은 여러 개 등록 가능하며 overlay에 여러 줄로 표시됩니다.
- 방송시간 타이머는 station_control.html의 방송시작/방송종료 버튼 기준으로 동작합니다.
- master.html에 방송국 삭제 버튼과 2단계 확인을 추가했습니다.
- 토큰 재생성도 2단계 확인을 추가했습니다.
- 공지/펀딩/노래방 텍스트는 좌측 정렬로 보정했습니다.


## 방송별 타이머 수정

- 방송시작/방송종료 버튼을 방송국 컨트롤의 방송 목록 각 행 안으로 이동했습니다.
- 각 방송별로 타이머 상태가 따로 저장됩니다.
- 방송종료 후 같은 방송에서 다시 방송시작을 누르면 00:00:00으로 리셋하지 않고 기존 누적 시간부터 이어집니다.
- 오버레이는 현재 선택된 방송의 누적 타이머를 표시합니다.


## 노래방/펀딩/ALERT 정리 수정

- 펀딩 선택을 별도 셀렉트에서 제거하고 처리 프리셋 드롭다운에 `펀딩: 제목`으로 표시합니다.
- 펀딩 선택 시 후원 입력은 기존처럼 저장되고, 선택한 펀딩에 만원 단위로 자동 반영됩니다.
- 노래방 관리 영역은 admin.html에 있으며, control.html에서 노래방창 OFF면 admin에서도 숨겨집니다.
- control.html에서 노래방 데이터를 덮어써서 초기화되는 문제를 방지했습니다.
- 노래방 입력칸 크기를 키웠습니다.
- 노래방 오버레이를 이미지 참고 형태로 정리했습니다.
- 현재 코인보유는 하단 흐르는 텍스트로 닉네임-코인현황 형태로 표시합니다.
- ALERT가 좌측 끝에서 잘리는 문제를 줄이기 위해 overlay-stack 기준으로 위치를 다시 계산합니다.


## 공지/펀딩/노래방 공통 데이터화

- 새 방송을 시작해도 공지, 펀딩, 노래방 내역이 이전 방송에서 이어지도록 수정했습니다.
- 방송별로 따로 저장되는 항목에서 공지/펀딩/노래방을 제외했습니다.
- 방송시간 타이머는 방송별로 유지됩니다.
- 후원 데이터는 방송별로 유지됩니다.


## 오류 수정

- control.html 펀딩 저장 시 `Cannot read properties of null (reading 'value')` 오류를 방지했습니다.
- control.html에서 노래방 데이터를 덮어쓰지 않도록 보강했습니다.
- admin.html에서 노래방 저장 후 최신 데이터를 다시 불러오도록 수정했습니다.
- overlay 노래방 박스 상단 작은 핑크 글씨를 제거했습니다.
- 예약 없음은 예약 리스트가 비어 있을 때만 표시되도록 정리했습니다.
