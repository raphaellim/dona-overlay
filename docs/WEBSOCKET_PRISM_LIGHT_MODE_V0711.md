# WebSocket / PRISM 경량 모드 V0711

## 변경 사항

- `server.js`가 Socket.IO 서버를 같은 HTTP 포트에서 실행합니다.
- 성공한 API 입력·수정·삭제 후 해당 방송국 room에 `overlay:changed` 이벤트를 전송합니다.
- `overlay.html`은 최초 연결 시 한 번 전체 상태를 동기화합니다.
- 연결 중에는 기존 1초 전체 폴링을 사용하지 않습니다.
- 짧은 시간에 여러 변경이 발생하면 120ms 동안 묶어 한 번만 갱신합니다.
- 데이터 요청 진행 중 추가 이벤트는 대기시킨 후 한 번만 재조회합니다.
- WebSocket 연결이 끊어진 경우에만 15초 간격 백업 조회를 수행합니다.
- 룰렛은 연결 중 0.5초 폴링을 중단하고 룰렛 이벤트가 있을 때만 확인합니다.
- 룰렛 백업 조회는 WebSocket 연결 장애 시에만 5초 간격으로 수행합니다.
- WebSocket 압축을 꺼서 모바일 프리즘의 CPU 사용을 줄였습니다.
- 후원·합계만 변경된 경우 상단 슬라이드 DOM을 다시 생성하지 않습니다.
- 이미지 목록·간격·ON/OFF가 실제로 변경된 경우에만 슬라이드를 다시 구성합니다.
- 슬라이드 OFF 또는 목록 변경 시 기존 이미지·영상의 `src`, 이벤트 핸들러와 재생 참조를 해제합니다.
- ALERT 종료 시 `show` 제거뿐 아니라 `hidden`, 인라인 표시 상태와 내용을 함께 정리해 프리즘에서 화면이 남는 현상을 방지합니다.

## 배포

기존 프로젝트와 동일하게 전체 파일을 배포하면 됩니다. `package.json`과
`package-lock.json`에 Socket.IO가 포함되어 있으므로 배포 환경에서 의존성이 자동 설치됩니다.

오버레이 주소와 `station`, `token`, `monitor` 파라미터는 기존 주소를 그대로 사용합니다.

## 장애 대응

WebSocket을 사용할 수 없는 네트워크에서는 Socket.IO가 보조 전송 방식을 시도합니다.
연결이 완전히 끊겨도 오버레이는 15초 백업 동기화로 현재 상태를 복구합니다.
