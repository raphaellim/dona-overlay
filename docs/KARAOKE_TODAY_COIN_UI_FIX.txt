# KARAOKE_TODAY_COIN_UI_FIX

수정 목적:
- 노래방 관리 카드 배치가 어색하게 보이는 문제 보정
- 코인내역에 과거 전체 보유자/이전 방송 내역이 섞여 보이는 문제 보정
- 코인내역을 금일/현재 방송 기준으로 이월, 충전, 사용 내역만 보이게 변경

수정 파일:
- server.js
- public/admin.html
- public/control.html

주요 수정:
1. server.js
- normalizeKaraokeData()에 carryoverUsers, coinLogs 보존 추가
- effectiveKaraokeData()에서 현재 방송의 carryoverUsers/coinLogs를 병합해서 내려주도록 수정
- /api/karaoke-data 저장 시 현재 방송 최초 이월 코인 스냅샷을 유지하도록 수정
- 다음 방송 생성/이월 시 coinLogs는 초기화, 남은 코인 users만 이월 유지

2. public/admin.html
- 노래방 카드 안의 코인추가/선곡 패널 배치 보정
- 코인내역 UI를 금일 기준 로그로 변경
- 코인내역 표시 기준: 이월 / 충전 / 사용
- 코인추가 시 충전 로그 기록
- 선곡 등록 시 사용 로그 기록
- 코인 수동수정 시 증가분은 충전, 감소분은 사용 로그 기록

3. public/control.html
- 컨트롤 페이지에서 코인추가/선곡 등록 시에도 동일한 coinLogs가 기록되도록 보강

동작 기준:
- 같은 방송 중 새로고침/ON-OFF: 노래 리스트와 금일 코인로그 유지
- 다음 방송 생성: 노래 리스트/예약/진행/완료/코인로그 초기화
- 다음 방송 생성: 남은 코인만 이월
- 코인내역 화면: 현재 방송의 이월 코인 + 오늘 발생한 충전/사용만 표시
