CREATOR CLIENT GATE DISABLE FIX

문제:
- m_creator.html은 서버 accessGuard에서 이미 방송국관리자/크리에이터 권한을 검사한다.
- 그런데 페이지 내부 JS가 /api/settings 또는 기타 API 응답의 role/권한 오류를 다시 해석하면서 로그인 게이트를 띄웠다.
- 그 결과 로그인은 성공했는데 화면 하단에 불러오는중 표시 후 다시 크리에이터 로그인 필요 창으로 돌아가는 루프가 생겼다.

수정:
- public/m_creator.html 내부 handleError()에서 권한/비밀번호 문구만으로 showLoginGate()를 호출하지 않게 변경.
- init() catch에서도 showLoginGate()를 다시 띄우지 않고 오류 ALERT만 표시하게 변경.
- m_creator.html 접근 권한은 server.js의 accessGuard에서만 처리한다.

정상 흐름:
- 미로그인 상태: server.js가 /station_login.html 로 이동시킴.
- 방송국관리자 로그인 후: m_creator.html 페이지가 열리면 내부 JS 오류로 다시 로그인 게이트를 띄우지 않음.
