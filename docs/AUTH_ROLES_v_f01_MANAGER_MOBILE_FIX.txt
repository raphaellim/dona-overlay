권한 구조 수정 요약

1. 최고관리자
- 전체 방송국 / 전체 페이지 / 전체 설정 권한.

2. 방송국관리자 = 크리에이터
- 해당 방송국 전체 권한.
- 방송 생성/삭제/수정 가능.
- m_admin.html, m_control.html 전체 기능 사용.
- m_creator.html은 크리에이터 리모컨으로 사용.

3. 방송매니저
- 당일 방송 비밀번호 기준.
- 기존 모바일 페이지를 제한모드로 사용.
- m_admin.html: 후원 입력, 최근 입력 확인/삭제, 용돈 조정 가능.
- m_control.html: 표시 ON/OFF, 공지, 펀딩 금액/목표/표시, 노래방, 미션타이머 등 당일 설정 가능.
- m_roulette.html: 수동 룰렛 실행/연속 실행/결과 숨김/결과 확인 가능.
- 방송 생성/삭제/과거 방송 선택/전체 합산/크리에이터별 합산/도네이터별 합산/크리에이터 목록 설정/프리셋 목록 설정/룰렛 목록 생성삭제/오버레이 토큰/미디어 관리는 제한.

구현 방식
- 모바일 페이지 파일은 하나로 유지하고 role 값에 따라 UI를 숨김 처리.
- 서버 API도 role 기준으로 차단.
- 방송매니저는 /api/summary, /api/donations 전체 조회 차단.
- 방송매니저 최근 입력은 /api/manager/recent 사용.
- 방송매니저의 fundingData 저장은 기존 펀딩 ID 기준 병합하여 항목 추가/삭제를 방지.
