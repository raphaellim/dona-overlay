# CREATOR LOGIN GATE ROLLBACK FIX

기준:
- dona-overlay-main_0608_v_f01_creator_login_loop_fix.zip 기반
- m_creator.html 로그인 흐름은 dona-overlay-main_0608_v_f01_karaoke_persist_fix.zip 당시처럼 되돌림

문제:
- 다른 페이지는 방송국관리자 로그인 후 정상 진입됨
- /m_creator.html?station=duugi 만 로딩 후 다시 '크리에이터 로그인 필요' 게이트로 돌아감
- 원인: m_creator.html 내부에서 /api/creator-auth 와 /api/settings role 판정을 이중으로 확인하면서 모바일/브라우저 쿠키 판정 타이밍이 어긋날 때 클라이언트 게이트가 다시 뜸

수정:
1. m_creator.html은 서버 accessGuard에서 이미 방송국관리자/크리에이터만 통과하므로, 페이지 내부에서 /api/creator-auth 이중 검사를 하지 않게 변경
2. loadAll() 흐름을 karaoke_persist_fix 시점과 동일하게 /api/station-info → /api/settings 순서로 복구
3. 페이지가 이미 열렸는데 /api/settings role이 늦게 guest/broadcast_manager로 들어와도 클라이언트 게이트를 다시 띄우지 않도록 보정
4. init()에서 다시 로그인 게이트를 띄우는 분기를 제거

기대 동작:
- 미로그인 상태: 서버에서 /station_login.html 로 이동
- 방송국관리자 로그인 성공 후: /m_creator.html?station=duugi 정상 표시
- 방송매니저 로그인: 서버에서 m_creator.html 접근 불가, 기존처럼 m_admin/m_control/m_roulette 제한모드 사용
